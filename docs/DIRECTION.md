## 海外モデルCをGitHub版に翻訳

```
[Gumroad中心]  作る → 出品 → 買ってもらう
[GitHub中心]  公開する → 使われる → 信頼が溜まる → 収益化
```

| 段階 | GitHubでやること | 収益 |
|------|-----------------|------|
| 1. 信頼 | 無料キット公開、README完璧、デモ動く | $0 |
| 2. 拡散 | 技術記事1本、「このキットで作った」スクショ | $0 |
| 3. 収益化 | Sponsors / 有料第2キット / 実装相談 | $29〜/月〜 |

海外の参考: Edwin Klesman（Cursor拡張・Gumroad $148）は小さいが、「自分の困りごと→キット」の証明。Bonu（SiteGPT）は**無料ツール量産→本番**だが、開発者キットなら**GitHub OSSが無料ツールの役割**を担う。

## 収益化の順番（急がない）

1. **今** — GitHubで無料版を価値あるものにする（方向決めフェーズ）
